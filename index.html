Algoritmo de Kruskal

Descobrindo a Árvore Geradora Mínima através de um algoritmo guloso elegante e eficiente

Introdução

O Algoritmo de Kruskal é uma das joias da teoria dos grafos, desenvolvido pelo matemático Joseph Kruskal em 1956. Este algoritmo resolve um dos problemas fundamentais da ciência da computação: encontrar a Árvore Geradora Mínima (AGM) de um grafo conexo e ponderado.

O que é uma Árvore Geradora Mínima?

Uma Árvore Geradora Mínima é um subconjunto das arestas de um grafo que conecta todos os vértices sem formar ciclos, minimizando o peso total das arestas selecionadas. É como encontrar a maneira mais econômica de conectar todas as cidades de um país com estradas, sem criar rotas redundantes.

O algoritmo de Kruskal é classificado como um algoritmo guloso (greedy), pois em cada etapa ele faz a escolha localmente ótima - seleciona sempre a aresta de menor peso disponível que não forme um ciclo. Surpreendentemente, essa estratégia "gananciosa" garante a solução globalmente ótima.

Como Funciona o Algoritmo

1
Inicialização

Crie uma floresta onde cada vértice do grafo é uma árvore individual. Inicialize uma estrutura Union-Find para detectar ciclos eficientemente.

2
Ordenação das Arestas

Ordene todas as arestas do grafo em ordem crescente de seus pesos. Esta é a etapa que determina a complexidade temporal do algoritmo.

3
Seleção Gulosa

Para cada aresta na lista ordenada, verifique se ela conecta dois componentes diferentes. Se sim, adicione-a à AGM e una os componentes.

4
Detecção de Ciclos

Use a estrutura Union-Find para verificar eficientemente se adicionar uma aresta criaria um ciclo. Se criaria, descarte a aresta.

5
Conclusão

O algoritmo termina quando temos exatamente n-1 arestas (onde n é o número de vértices) ou quando todas as arestas foram consideradas.

Análise de Complexidade

A eficiência do Algoritmo de Kruskal depende principalmente de duas operações: a ordenação das arestas e as operações Union-Find para detecção de ciclos.

Operação Complexidade Descrição
Ordenação das Arestas O(E log E) Onde E é o número de arestas
Union-Find (com otimizações) O(E α(V)) Onde α é a função inversa de Ackermann
Complexidade Total O(E log E) Dominada pela ordenação
Espaço O(V) Para a estrutura Union-Find

Como E ≤ V², temos que log E = O(log V), então a complexidade também pode ser expressa como O(E log V).

Implementação

função Kruskal(G): // Inicialização A = conjunto vazio // Árvore geradora mínima // Criar conjuntos disjuntos para cada vértice para cada vértice v em G.V: FAZER-CONJUNTO(v) // Ordenar arestas por peso crescente ordene as arestas de G.E por peso w // Processar cada aresta para cada aresta (u, v) em G.E (ordenadas): // Verificar se u e v estão em componentes diferentes se ENCONTRAR-CONJUNTO(u) ≠ ENCONTRAR-CONJUNTO(v): A = A ∪ {(u, v)} // Adicionar aresta à AGM UNIR(u, v) // Unir os componentes // Parar se temos n-1 arestas se |A| = |G.V| - 1: quebrar retornar A

Comparação com Outros Algoritmos

O Algoritmo de Kruskal não é o único método para encontrar árvores geradoras mínimas. Vamos compará-lo com seu principal concorrente:

Kruskal vs. Prim

Algoritmo de Prim: Constrói a AGM crescendo uma única árvore, sempre adicionando a aresta de menor peso que conecta um vértice na árvore a um vértice fora dela.

Algoritmo de Kruskal: Constrói a AGM considerando arestas globalmente, mantendo uma floresta de árvores que gradualmente se unem.

Quando usar cada um: Prim é mais eficiente para grafos densos, enquanto Kruskal é preferível para grafos esparsos.
